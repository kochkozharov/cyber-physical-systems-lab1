# Лабораторная работа 1: Детекция объектов — Кошки и Собаки

## 1. Датасет и обоснование выбора

**Asirra: Cats vs Dogs** — 1 100 изображений кошек + 1 100 изображений собак, каждое снабжено аннотацией в формате Pascal VOC XML с ограничивающим прямоугольником (bounding box) вокруг животного.

**Задача:** детекция одного класса (метка = `animal`) без разделения на кошка/собака.

Для того чтобы обучение было быстрее отобраны 300 изображений (150 кошек + 150 собак), разбитых в соотношении 70 / 20 / 10 (train / val / test).

## 2. Метрики качества

| Метрика | Обоснование |
|---|---|
| **mAP@0.5** | Основная стандартная метрика детекции: площадь под кривой precision-recall при IoU ≥ 0.5. |
| **mAP@0.5:0.95** | Более строгая метрика, усредняющая результаты по порогам IoU от 0.5 до 0.95 с шагом 0.05; штрафует за неточную локализацию. |
| **Precision** | Доля предсказанных рамок, которые верны — важна, когда дорого ошибиться ложным срабатыванием. |
| **Recall** | Доля реальных объектов, которые были найдены — важна, когда дорого пропустить объект. |

## 4. Скрипты

### `prepare_dataset.py`
Конвертирует аннотации Pascal VOC XML в формат YOLO (один класс, нормализованные координаты), отбирает 300 изображений, разбивает на train/val/test и записывает `data/dataset.yaml`.

```bash
python prepare_dataset.py
```

Вывод:
```
Dataset prepared: 210 train / 60 val / 30 test
Config: data/dataset.yaml
```

---

### `train_baseline.py`
Обучает две базовые модели из семейства ultralytics YOLOv11:
- **yolo11n** — чисто свёрточный backbone (YOLOv11 nano, CNN).
- **rtdetr-l** — трансформерный детектор (RT-DETR large), доступный через ultralytics.

Обе модели обучаются **5 эпох** с гиперпараметрами по умолчанию.
Скрипт устанавливает `PYTORCH_ENABLE_MPS_FALLBACK=1` внутри, чтобы оператор `grid_sampler_2d_backward` у RT-DETR откатывался на CPU на Apple Silicon.

```bash
python train_baseline.py
```

Веса сохраняются в `runs/baseline/{model}/weights/best.pt`.

---

### `evaluate.py`
Оценивает сохранённые веса модели на **тестовой выборке** и выводит таблицу метрик.

```bash
# Все доступные модели:
python evaluate.py

# Конкретные модели:
python evaluate.py yolo11n_baseline rtdetr_l_baseline
```

Доступные ключи моделей: `yolo11n_baseline`, `rtdetr_l_baseline`, `yolo11n_improved`, `rtdetr_l_improved`.

---

### `train_improved.py`
Переобучает обе модели с улучшенной конфигурацией на основе следующих гипотез:

| Гипотеза | Изменение | Ожидаемый эффект |
|---|---|---|
| Более сильная аугментация | mosaic=1.0, mixup=0.15, HSV-джиттер, поворот, отражение | Лучшая обобщающая способность |
| Косинусный LR + более долгий прогрев | cos_lr=True, warmup_epochs=3 | Более стабильная сходимость |
| Weight decay + label smoothing | weight_decay=5e-4, label_smoothing=0.05 | Снижение переобучения на малом датасете |

```bash
python train_improved.py
```

Веса сохраняются в `runs/improved/{model}/weights/best.pt`.

---

### `custom_detector.py`
Реализует детектор одного класса **с нуля** на PyTorch:

**Архитектура:**
- Backbone: MobileNet-подобная CNN с depthwise-separable свёртками (stem + 6 DW-блоков → 512-мерный вектор через global average pooling).
- Голова: двухслойный MLP, предсказывающий `(conf, cx, cy, w, h)` — anchor-free, одномасштабный.
- Функция потерь: BCE для уверенности + SmoothL1 для координат рамки.

Обучает два варианта: без аугментации и с аугментацией (цветовой джиттер, случайное отражение, случайный поворот). Использует то же разбиение train/val/test, что и эксперименты с YOLO.

```bash
python custom_detector.py
```

Веса сохраняются в `runs/custom/{variant}/best.pt`.

---

## 5. Воспроизведение всех результатов (end-to-end)

```bash
# 1. Подготовка данных
python prepare_dataset.py

# 2. Базовый бейзлайн
python train_baseline.py
python evaluate.py yolo11n_baseline rtdetr_l_baseline

# 3. Улучшенный бейзлайн
python train_improved.py
python evaluate.py yolo11n_improved rtdetr_l_improved

# 4. Полное сравнение всех моделей
python evaluate.py

# 5. Кастомный детектор
python custom_detector.py
```

## 6. Результаты

Оценка на тестовой выборке из 30 изображений (seed=42). mAP@0.5 кастомного детектора вычисляется как доля предсказаний с IoU ≥ 0.5 относительно разметки.

### Базовый бейзлайн (Раздел 2)

| Модель | Тип | P | R | mAP@0.5 | mAP@0.5:0.95 |
|---|---|---|---|---|---|
| yolo11n | CNN | 0.879 | 0.778 | 0.861 | 0.549 |
| rtdetr-l | Трансформер | 0.865 | 0.892 | 0.875 | 0.611 |

### Улучшенный бейзлайн (Раздел 3)

| Модель | Тип | P | R | mAP@0.5 | mAP@0.5:0.95 | Δ mAP@0.5 к базовому |
|---|---|---|---|---|---|---|
| yolo11n | CNN | 0.850 | 0.472 | 0.680 | 0.331 | **−0.181** |
| rtdetr-l | Трансформер | 0.883 | 0.806 | 0.823 | 0.420 | −0.052 |

**Выводы (Раздел 3й):**
- «Улучшенная» конфигурация **ухудшила** обе модели. Причина: `warmup_epochs=3` при всего 5 эпохах означает, что 60% обучения уходит на прогрев — модель едва достигает пикового learning rate перед остановкой. Кроме того, `lr0=0.005` (вдвое ниже дефолтного) в сочетании с косинусным расписанием даёт недостаточный обучающий сигнал при столь коротком обучении.
- RT-DETR устойчивее к этим гиперпараметрам (крупная модель, сильные pretrain-приоры) и теряет лишь 5 п.п. mAP@0.5.
- **Главный вывод:** аугментации и расписания LR, рассчитанные на 50–300 эпох, вредят при обучении всего 5 эпох. Дефолтные гиперпараметры ultralytics настроены именно под короткий fine-tuning и не требуют изменений.
- RT-DETR показывает более высокий recall (0.892 против 0.778) и лучший mAP@0.5:0.95 (0.611 против 0.549), демонстрируя превосходящее качество локализации ценой ~10× большего времени обучения.

### Кастомный детектор (Раздел 4)

Реализован с нуля: MobileNet-подобный depthwise-separable backbone → global average pooling → MLP-голова, предсказывающая `(conf, cx, cy, w, h)`. Функция потерь: BCE + SmoothL1.

| Модель | P | R | mAP@0.5 |
|---|---|---|---|
| custom (без аугментации) | 0.800 | 0.800 | 0.800 |
| custom (с аугментацией) | 0.800 | 0.800 | 0.800 |

**Выводы (Раздел 4й):**
- Кастомный детектор достигает mAP@0.5 = 0.800, что неожиданно конкурентоспособно по сравнению с YOLO (0.861). Отчасти это объясняется простотой задачи: одиночные крупные животные по центру кадра с чёткими bounding box'ами.
- Аугментация не изменила результат на тесте — вероятно, 5 эпох недостаточно для того, чтобы аугментация успела дать регуляризационный эффект (модель ещё не переобучилась настолько, чтобы это имело значение).
- Кастомная модель заведомо проигрывает по точной локализации (нет feature pyramid, одномасштабная голова) — что отразилось бы в mAP@0.5:0.95, если бы метрика считалась аналогично.
- По сравнению с улучшенным бейзлайном: кастомная модель превзошла yolo11n-improved (0.800 против 0.680), что демонстрирует — чрезмерная подстройка гиперпараметров способна навредить предобученной модели сильнее, чем простая реализация с нуля.
- Для продакшена предпочтительны модели ultralytics (многомасштабная детекция, anchor-логика, продвинутый NMS); кастомная реализация служит образовательным бейзлайном, демонстрирующим механику пайплайна детекции.
